# Сводка: проект "Читалка глав Shadow Slave / Теневой Раб"

## Контекст задачи
Пользователь читает веб-новеллу «Теневой Раб» (Shadow Slave). Главы 1–1035
доступны на Ranobelib, а начиная с главы 989 (частичное пересечение) и дальше
(до ~3000+, онгоинг) перевод публикуется только на telegra.ph, без единого
списка/пагинации на самом сайте. Задача — автоматизировать сбор и чтение этих
глав, в идеале в виде отдельного приложения на Android (у пользователя только
телефон, компьютера нет).

## Ключевые технические находки
- У каждой главы на telegra.ph свой URL-слаг (заголовок+дата), общего паттерна
  нумерации нет — просто перебрать номера нельзя.
- Переводчик ведёт **страницы-оглавления по томам**, например:
  `https://telegra.ph/5-tom-989-1060-04-29` — список глав 989–1060.
  В конце каждой такой страницы — ссылка **«Следующий том»**, ведущая на
  следующее оглавление (6 том, 7 том и т.д.). Это и есть надёжный способ
  обойти вообще все главы, не полагаясь на то, есть ли навигация внутри
  самих глав (внутри глав ссылки "Следующая глава" появляются только начиная
  примерно с главы ~1839 и позже — то есть непоследовательно, полагаться на
  них нельзя).
- Главная страница-навигатор по всей новелле:
  `https://telegra.ph/Navigaciya-po-novelle-Tenevoj-Rab-10-02`
- Есть открытый **Telegraph API**: `https://api.telegra.ph/getPage/{slug}?return_content=true`
  — отдаёт JSON с текстом страницы (заголовок + дерево контент-нод). У API
  открыт CORS (`Access-Control-Allow-Origin: *`), работает даже без токена.

## Сделанные инструменты (в хронологическом порядке)

### 1. Python-скрипт `download_chapters.py`
Скачивает все главы через Termux (терминал на Android, без компиляции).
- Проходит по цепочке томов через «Следующий том», собирает {номер главы: url}.
- Качает текст каждой главы через Telegraph API, сохраняет в `chapters/NNNNN.txt`.
- Есть retry с нарастающей паузой (до 4 попыток) на случай обрывов SSL/сети
  (были ошибки `SSLEOFError` — типично для нестабильного VPN-соединения).
- Идемпотентен: при повторном запуске пропускает уже скачанные файлы.
- Установка в Termux: `pkg install python` → `pip install requests beautifulsoup4`.
- Файл скачивался в обычный браузер → переносился в Termux командой
  `cp ~/storage/downloads/download_chapters.py ~/` (после `termux-setup-storage`).

### 2. HTML-загрузчик `shadow_slave_downloader.html` (промежуточный вариант)
Веб-версия того же самого, работающая через fetch() в браузере.
**Проблема**: при открытии файла напрямую с диска (`file://`) мобильный Chrome
блокирует все fetch-запросы наружу (даже к самому telegra.ph) — это ограничение
безопасности для страниц без нормального https-origin, а не блокировка сети/VPN
(это было тщательно продиагностировано: VPN исправен, обычная навигация в
Chrome на telegra.ph работает, но JS fetch() с локального файла — нет).
Отсюда и родилась идея перейти к варианту с полноценным хостингом (см. ниже).

### 3. HTML-читалка `shadow_slave_reader.html` (промежуточный вариант)
Отдельная читалка, работающая с папкой `chapters/`, которую создаёт питон-скрипт:
- Выбор папки через `<input type="file" webkitdirectory>`.
- Список глав, пометки прочитано/не прочитано, кнопки Пред./След., регулировка
  размера шрифта.
- Прогресс не мог сохраняться автоматически (та же причина: file:// не даёт
  надёжно использовать localStorage) — поэтому был сделан ручной экспорт/импорт
  прогресса через маленький JSON-файл.

### 4. PWA-приложение (текущий, финальный вариант) — папка `pwa/`
Объединяет скачивание и чтение в одно целое, рассчитано на размещение по
нормальному https-адресу (не file://), тогда работают и fetch(), и постоянное
хранилище браузера. Состоит из 5 файлов:
- **index.html** — всё приложение (сбор списка глав через Telegraph API,
  скачивание текста, чтение, listing, регулировка шрифта).
  - Список глав (номер→url) хранится в `localStorage` (`ss_index`).
  - Прогресс чтения (какие главы прочитаны, номер последней) — тоже в
    `localStorage` (`ss_progress`), сохраняется автоматически.
  - Сам текст скачанных глав хранится в **IndexedDB** (база `shadowSlaveDB`,
    хранилище `chapters`) — это и даёт офлайн-чтение после первого скачивания.
  - Индикатор "онлайн/офлайн" в шапке.
- **manifest.json** — манифест PWA (имя «Хроники: Теневой Раб», standalone-режим,
  иконки).
- **sw.js** — service worker: кэширует оболочку приложения (html/css/js/иконки)
  для офлайн-запуска; запросы к api.telegra.ph намеренно не кэширует (они либо
  идут в сеть, либо падают, если сети нет — тогда доступны только уже
  скачанные в IndexedDB главы).
- **icon-192.png**, **icon-512.png** — сгенерированные иконки (тёмный фон,
  зелёный ромб-глиф).

## План сборки в APK (согласован с пользователем, вариант 2 из предложенных)
1. **GitHub Pages**: создать репозиторий на github.com, загрузить туда все 5
   файлов из папки `pwa/` в корень репозитория, включить Pages в Settings
   (Source: ветка main, папка /root) → получить https-адрес вида
   `https://ник.github.io/repo-name/`.
2. **PWABuilder** (pwabuilder.com, официальный сервис от Microsoft): вставить
   туда ссылку на GitHub Pages, нажать Start — сервис проверит manifest и
   service worker и соберёт `.apk`/`.aab` прямо в браузере, без установки
   каких-либо SDK. Скачать готовый файл, установить на телефон (один раз
   разрешить установку из неизвестных источников).
3. Уточнение по ходу: если случайно закрыть/выйти с сайта PWABuilder — ничего
   не теряется, файлы на GitHub Pages никуда не деваются, нужно просто заново
   зайти на pwabuilder.com и повторно вставить ту же ссылку.

## Статус на момент сводки
Пользователь ещё не завершил загрузку файлов на GitHub / сборку APK — это
следующий шаг. Все 5 файлов PWA уже сгенерированы и отправлены пользователю
в чате (переписать заново не нужно, они готовы к загрузке на GitHub Pages).

- Помощь с настройкой GitHub Pages.

- Возможные доработки самого приложения: например, кнопка "скачать все главы
  разом" с ограничением по батчам, тёмная/светлая тема, синхронизация прогресса
  между устройствами и т.д. — на момент сводки не запрошены, но могут всплыть.
